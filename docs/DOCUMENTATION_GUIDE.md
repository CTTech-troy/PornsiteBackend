# API Documentation Automation

The documentation in `API_DOCUMENTATION.md` is automatically generated by scanning the source code and database migrations.

## How to regenerate documentation

If you have added new routes, database tables, or WebSocket events, you can update the documentation by running:

```bash
# Ensure you are in the root directory
python3 scripts/doc_generator.py
```

## Tools used

The generation process uses three specialized scanners located in the `scripts/` directory:

1.  **`api_scanner.py`**: Scans `src/router/` to detect all API endpoints, methods, and authentication requirements.
2.  **`db_scanner.py`**: Scans `supabase/migrations/` to extract the database schema, including tables, columns, and RLS policies.
3.  **`ws_scanner.py`**: Scans `index.js` to map all Socket.IO incoming and outgoing events.

The `doc_generator.py` script orchestrates these scanners and compiles the results into a clean Markdown format.

## Requirements

- Python 3.x
- No additional dependencies required (uses standard libraries)
